了解です。以下は、**各項目をYES/NOで判断できる**ように再構成し、さらに

* **YESなら何を仮定するか（モデルに何を入れるか）**
* **NOなら何を仮定するか（何を無視/単純化するか）**
* **YES/NOで表現できる現象が何か**
* 用語をわかりやすく（例：

  * *ドレイン* → **水切り（引き上げ後の液切り・排液時間）**
  * *バス* → **薬液槽（タンク）**
  * *補給* → **薬液の入れ替え/追加（リフレッシュ）**
  * *循環* → **槽内循環（ポンプ循環・攪拌）**
    ）

…という形で整理しました。

---

# 0) 使い方（現場での運用）

1. 下の表を工程ごとに **YES/NO**（迷う場合は “一旦YES”）で埋める
2. YESの項目に対応する **モデルモジュール（近似）**を有効化
3. NOの項目は **単純化（一定/無視/均一）**して実装コストを下げる
4. 実測との残差（ムラ形状）で、YES項目を追加していく

---

# 1) 反応・表面化学（界面）チェック（YES/NO）

> 目的：膜厚変化（エッチ/成長）が「化学反応側」にどれだけ支配されるかを判断

|  # | YES/NOで答える確認事項（工程エンジニアが判断できる形）                    | YESなら：仮定・モデル化（何を入れる？）                              | NOなら：仮定・モデル化（何を単純化？）        | YESで表現できる現象（何が再現できる？）        |
| -: | ------------------------------------------------- | -------------------------------------------------- | --------------------------- | ---------------------------- |
| R1 | **温度を±2〜5℃変えると、膜厚変化（nm）がはっきり変わる**（同じ条件で）          | 反応速度に温度依存を入れる：(k(T)=k_0\exp(-E_a/RT))              | 温度は一定（代表温度）として扱う／温度依存は係数に吸収 | 温度ムラや日変動によるレート差の説明、温度設計の感度評価 |
| R2 | **薬液濃度（主成分）を変えると、膜厚変化が比例（線形）ではない**                | 反応次数 (n) を持たせる：(r=kC_s^n)                          | (n=1)（線形）で固定                | 濃度変更で効きが急増/鈍化する挙動の説明         |
| R3 | **濃度を上げても、ある濃度以上でレートが頭打ち**になる                     | 吸着飽和（Langmuir）を入れる：(r=k\frac{KC_s}{1+KC_s})        | べき乗（または線形）で十分               | 高濃度側の飽和、最適濃度の設計              |
| R4 | **同じレシピでも、薬液槽（タンク）の使用が進むとレートが落ちる/変わる**（同一温度・流量でも） | 反応阻害・生成物影響を入れる（阻害項 or “汚れ指標”）                      | “新液同等”として一定扱い               | バス寿命末期の低下、ロット間ばらつき、寿命管理の定量化  |
| R5 | **洗浄中に膜が増える/薄膜が残る**傾向がある（SC1/SC2/O₃水などで）          | “成長＋溶解”競合ODEを入れる：(\dot{x}=G(C_{ox})-E(C_{etch},x)) | 「膜は減るだけ」or「膜は一定」の単純モデル      | 初期成長→飽和、条件で残膜が変わる現象の説明       |
| R6 | **前工程の表面状態（HF last/酸化膜有無/汚染状態）で初期レートが変わる**        | 初期状態パラメータ（初期膜厚/被覆率など）を入力として扱う                      | 初期状態は同一と仮定（初期条件固定）          | “立ち上がりだけ違う”現象、ロット間の初期差の吸収    |

# 2) 物質移動・流れ（境界層/カセット内）チェック（YES/NO）

> 目的：位置ムラ（上段下段、端/中央）が「輸送側」で決まるかを判断
> キーワード：**境界層（液膜）**、**循環（攪拌）**、**ウェハ間の流れ**

|  # | YES/NOで答える確認事項                 | YESなら：仮定・モデル化                                     | NOなら：仮定・モデル化       | YESで表現できる現象                |
| -: | ------------------------------ | ------------------------------------------------- | ------------------ | -------------------------- |
| T1 | **循環流量/攪拌条件を変えると膜厚変化が変わる**     | 境界層輸送（物質移動係数）を入れる：(J=k_m(C_b-C_s))。(k_m)は条件依存     | (k_m)一定（反応律速）として扱う | 流量を上げると進む/均一化する、攪拌設計の効果    |
| T2 | **上段/下段（スロット番号）で膜厚変化が系統的に違う**  | “位置ごとに違う”を持たせる：スロット別 (k_{m,i}) or スロット別 (C_{b,i}) | スロット差なし（全枚同一）      | スロット依存ムラ、枚数/ピッチの影響の説明      |
| T3 | **供給側/排出側（タンク内の位置）で差**が出る      | 槽内が一様ではない：**区画モデル（CSTR network）**で (C_j(t)) を持つ   | 槽内は完全混合（0D一様）      | “供給近傍が強い”など装置配置起因のムラ       |
| T4 | **中心/外周で同心円状のムラ**（外周が強い/弱い）    | 面内の (k_m(r)) または入力 (C_b(r)) を持つ（マップ化）             | 面内均一（1点代表）         | 面内ムラの再現、端部対策の評価            |
| T5 | **枚数を増やすとムラが増える／ピッチを詰めると遅くなる** | ウェハ間干渉（狭隙輸送）を反映：スロット別 (k_m) を下げる、必要なら狭隙モデル        | ウェハ間干渉なし           | 高密度搭載時の遅れ、設計限界評価           |
| T6 | **流量を増やしてもレートが上がらない**（飽和）      | 反応律速または吸着飽和（R3）優先。輸送の寄与は小さいと仮定                    | 物質移動モデルを簡略化        | “流量を上げても効かない”を説明、対策が化学側になる |

# 3) 薬液槽（タンク）状態の時間変化チェック（YES/NO）

> 目的：ロット順・時間経過で効きが変わるか（＝槽状態をモデルに入れるか）

|  # | YES/NOで答える確認事項                  | YESなら：仮定・モデル化                                                         | NOなら：仮定・モデル化  | YESで表現できる現象         |
| -: | ------------------------------- | --------------------------------------------------------------------- | ------------- | ------------------- |
| B1 | **処理の後半ほどレートが下がる/変わる**（同一ロット内）  | 槽濃度 (C(t)) を時間発展：(\dot{C}=-\frac{A}{V}\langle J\rangle + \text{入れ替え}) | 槽濃度一定（時間変化なし） | 長時間処理の“尻すぼみ”、最適処理時間 |
| B2 | **同じレシピでも、薬液を作ってからの経過時間で効きが違う** | 分解（自己劣化）を入れる：(\dot{C}=-k_{dec}C)                                      | 分解なし（新液一定）    | 待機時間の影響、立ち上げ手順の最適化  |
| B3 | **薬液の入れ替え/追加（リフレッシュ）で急に効きが戻る**  | 入れ替え（流入・排出）項を入れる：(\frac{Q}{V}(C_{in}-C))                              | 入れ替えなし        | 補給戦略の効果を定量化         |
| B4 | **寿命末期に欠陥/ムラが増える**（再付着・析出っぽい）   | 生成物/汚れ状態変数 (P(t)) を導入し、阻害or再付着を表現                                     | 生成物影響なし       | 寿命管理、フィルタ/循環の効果評価   |

# 4) 初期置換（DI水の持ち込み）・液切り（排液）・気泡チェック（YES/NO）

> 用語を明確にします

* **水切り（液切り）**：ウェハ引き上げ後、表面の液が切れるまでの時間・状態
* **置換**：DI水や前液が、処理液に置き換わる過程
* **気泡**：表面に付着して局所的に反応を遮る

|  # | YES/NOで答える確認事項                    | YESなら：仮定・モデル化                             | NOなら：仮定・モデル化    | YESで表現できる現象         |
| -: | --------------------------------- | ----------------------------------------- | --------------- | ------------------- |
| I1 | **投入直後（最初の数十秒）だけ挙動が違う**（遅い/ムラが出る） | 初期置換遅れを入れる：反応開始遅延 (\Delta t) または初期濃度が低い期間 | 初期から即定常（置換遅れなし） | “初期だけ遅い/縞が出る”の説明    |
| I2 | **水切り（液切り）時間を変えると結果が変わる**         | “前液持ち込み量”を状態/パラメータ化（希釈・置換）                | 持ち込み無し          | 水切り条件の最適化、ばらつき要因の特定 |
| I3 | **縞模様・筋状ムラ**が出る                   | 置換/濡れの空間差（位置ごとに遅延/濃度差）を入れる                | 置換は一様           | 縞ムラの再現、投入条件の感度      |
| I4 | **斑点状の未エッチ領域**が出る（点状欠陥）           | 気泡マスク：有効面積低下 (J_{eff}=(1-\theta_b)J)      | 気泡影響なし          | 斑点ムラ、超音波・脱気の効果      |
| I5 | **メガソニックON/OFFで膜減りが変わる**          | (k_m)倍率（ONでkm増）や気泡剥離効果を入れる                | 音響効果なし          | メガソニック条件最適化         |

# 5) YES/NO → “何を仮定するか”が一目で分かるまとめ（モデル選定用）

最後に、実装や議論に直結するよう「YESなら何をONにするか」を一覧化します。

| YESが付いたカテゴリ      | コード/モデルでONにするもの（例）                           | NOなら固定するもの（例）                |
| ---------------- | -------------------------------------------- | ---------------------------- |
| 反応温度依存（R1）       | `reaction.temperature_dependence: arrhenius` | `T=const`                    |
| 反応非線形（R2/R3）     | `reaction.type: powerlaw / langmuir`         | `reaction.type: first_order` |
| 成長＋溶解（R5）        | `model: oxide_growth_dissolution`            | `oxide: disabled`            |
| 輸送影響（T1）         | `transport: film`（kmを入力or推定）                 | `km: const`                  |
| スロット差（T2）        | `domain: batch_slots`（スロット別km/Cb）            | `slot_uniform: true`         |
| タンク内非一様（T3）      | `model: batch_compartment`（区画）               | `tank: well_mixed`           |
| タンク時間変化（B1〜B3）   | `tank.mass_balance: enabled`                 | `C_tank=const`               |
| 初期置換/持ち込み（I1〜I3） | `initial_exchange: enabled`（遅延/希釈）           | `initial_exchange: off`      |
| 気泡/音響（I4/I5）     | `bubble_masking / sonic_km_gain`             | `bubble/sonic: off`          |
