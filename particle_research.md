## 使い方のコツ（物理シミュ側の入力に落とすとき）

* **“発生位置分布”**を作りたい
  → (6)フレーク（応力/幾何でホットスポット）＋(3)(4)(5)再飛散（壁面せん断の強い所）
* **“初速度/放出角分布”**を作りたい
  → 低エネルギー境界条件なら (1)半Maxwell＋(\cos\theta)、イオン衝撃なら (2)Sigmund–Thompson＋角度分布
* **“時間変動する発生率（イベント）”**を入れたい
  → (3)(5)の resuspension rate 形式（(dN/dt=-pN)）が実装しやすい ([サイエンスダイレクト][4])

## 全体像：擬似生成は「物理を置き換える」のではなく「物理を速く・狙って・整理して回す」補助器

### 全体フロー（俯瞰）

```mermaid
flowchart LR
  A[擬似生成器 G(z)\n(多様なパターンを高速生成)] --> B[ターゲット/代表パターン選択\n(クラスタリング/境界重視)]
  B --> C[物理シミュレーション Phys(θ)\n(高コスト・因果ラベル付き)]
  C --> D[物理特徴量抽出\n(方向/半径/滞留/再衝突 etc.)]
  D --> E[整理・圧縮\nz_hatへフィット / 自動ラベリング]
  E --> F[次の物理DOE/探索方針更新\n(アクティブサンプリング)]
  C --> G[観測モデル H(·)\n検査の見え方を反映]
  G --> H[実測との比較・モデル校正\nPhysの改善/残差モデル]
  F --> B
```

## どれを選ぶ？目的別のおすすめセット（最短ルート）

| あなたのゴール                 | まず効く組み合わせ                 | 理由                            |
| ----------------------- | ------------------------- | ----------------------------- |
| 物理計算を**効率化**したい（高コスト対策） | **#2（アクティブDOE）＋ #3（逆問題）** | “狙って回す”＋“対応表を作る”で最短でデータ価値が上がる |
| 粒子追跡の**収束が悪い**/ノイズが大きい  | **#4（重要度サンプリング）**         | Monte Carloの分散を直接下げるので効果が出やすい |
| 物理モデルが**実測に似ない**        | **#7（観測モデル）＋ #5（残差）**     | 「検査の見え方」と「物理の不足」を分けて対処できる     |
| 物理結果が増えて**整理できない**      | **#1（z圧縮）＋ #6（自動ラベリング）**  | 物理を“画像の山”から“可検索な知識ベース”へ変換できる  |
