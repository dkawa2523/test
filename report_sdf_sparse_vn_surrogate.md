# 形状シミュレーション・サロゲート（1位案）詳細レポート  
## SDF narrow band + Sparse CNN による法線速度 \(V_n\) 閉包 + レベルセット更新 + 蒸留 + 同定

> 対象：半導体製造（主にエッチング／デポ／スパッタ／反射／表面拡散を含む）における、**Monte Carlo（粒子フラックス）× voxel表面**の3D形状シミュレーションを高速代替し、かつ実測SEMに合わせて未知パラメータ（フラックス／表面反応）を早期同定するためのサロゲートモデル。  
> 読者：データサイエンティスト／物理モデル開発者（両者が同じ図・同じ言葉で議論できることを狙う）。

---

## 0. エグゼクティブサマリー（要点）

### この手法の核
- 形状の時間発展を **3Dボクセルの直接予測**として学習するのではなく、  
  **界面の法線方向速度 \(V_n\)** を学習し、**レベルセット（SDF）**で幾何更新する。
- 「複雑なMC粒子反応を全部学習する」のではなく、**形状進化に必要な閉包（closure）**として  
  \(\;V_n = f(\text{局所幾何}, \text{レシピ条件}, \text{潜在パラメータ})\;\) を学習する。

### 目的に対する効き方（短く）
- **目的1：計算高速化**  
  - 高価なMC（2時間〜）を「学習データ生成」に限定し、運用は **SDF narrow band + Sparse CNN**で高速化。  
  - レベルセット更新は、半導体プロセス向け高性能実装の系譜があり、実装知見が蓄積（例：ViennaPSは高性能レベルセット＋階層run-length encoding）  
- **目的2：ブラックボックスパラメータ同定（実測SEMに合わせる）**  
  - 未知のフラックス・反応を**低次元潜在 \(z\)** に集約し、SEM特徴を観測として **同定（MAP/SBI）**する設計。  
  - シミュレータの尤度が書けない状況で、SBI（likelihood-free inference）を現実的に使える（`sbi reloaded`）。  
  - 学習時のみMC内部ログ（反射寄与など）を使う **Teacher–Student蒸留（LUPI/Distillation）**で、実運用（Student）は軽量・実測吸収に強い。

## 1. 背景・目的・課題（与件の整理）

### 背景
半導体製造のウェハ加工（エッチング/デポ/スパッタ等）では、微細形状の最終形状だけでなく**時間発展**（プロファイル変化）を捉える必要がある。しかし高忠実度の形状シミュレーション（例：粒子フラックス×voxel表面のMonte Carlo）は高価で、**条件探索やパラメータ同定**が困難になる。

### 目的
- **目的1**：形状シミュレーションに代わるサロゲートモデルにより、計算速度を大幅向上
- **目的2**：実測SEM形状に合うように、フラックス／表面反応などのブラックボックスパラメータを早期同定（物理的妥当範囲で）

### 現状・課題（重要ポイント）
- 物理：エッチ/デポ/スパッタ/反射/表面拡散などを含み、一次反応だけでなく**二次反応以降**も影響
- 計算：3Dで **2時間以上**、メモリも重い
- 実測：SEM画像（最終形状中心）。**マスク・丸み・微細形状**の合わせ込みが難しい
- 不足物理：温度依存など欠落があるかもしれないが、最終形状しか無く「何が足りないか不明」
- 装置情報：形状シミュレーション単体だと装置ノブと直結しない
- 実測条件数：工程ごと**10条件程度**
- 過去の失敗：プロセスごと1万条件以上の学習が必要／入力変数が多すぎ／運用コスト高／実測再現できず

## 2. 提案手法（1位案）の全体像

### 提案手法の名前（本レポートでの呼称）
**“SDF narrow band + Sparse CNN による \(V_n\) 閉包 + レベルセット更新 + 蒸留 + 同定”**

### 設計の基本原理（なぜ効くか）
1) **形状表現を voxel から SDF（Signed Distance Field）へ**  
   - 形状の連続性（滑らかさ）を担保しやすい  
   - 法線・曲率などの幾何量が計算可能  
   - レベルセット更新でトポロジ変化にも強い  
2) **SDFの“表面近傍（narrow band）だけ”を学習・推論対象にする**  
   - 形状進化に効く領域に計算資源を集中  
   - データ量・計算量を大幅削減  
3) **Dense 3D CNNではなく Sparse CNN（疎畳み込み）を使う**  
   - narrow bandの点だけをSparseTensorとして計算でき、メモリ効率が高い  
4) **学習ターゲットは形状そのものではなく法線速度 \(V_n\)**  
   - MCの複雑反応を「形状進化に必要な閉包」として吸収  
   - 推論は安定した幾何更新（レベルセット）へ渡す  
5) **学習時のみ“MC内部ログ”をTeacherに与え、Studentへ蒸留**  
   - 「内部影響（遮蔽/反射/再入射/二次反応）」を入れつつ、運用は軽量  
6) **実測SEMで未知を同定（latent \(z\) の推定）**  
   - 実測データが少なくても、「モデル本体を壊さず」合わせ込みが可能

## 4. 形状表現：Voxel → “表面中心”へ移す効果と、SDF採用理由

### 4.1 voxel直学習（過去方式）が抱えやすい問題
- 3D voxelの全体をU-Net/RNN等で予測  
  - **メモリ・計算**が重い（特に高解像）  
  - 出力が離散（0/1）で、**微妙な丸み・薄膜**が扱いにくい  
  - 時系列ロールアウトで誤差が蓄積し、形状が破綻しやすい  
  - “反射/再入射”など非局所効果は入力工夫が必要で、学習データが膨大になりがち

### 4.2 表面中心化の基本発想（Why surface-centric）
- 形状進化に本質的に必要なのは「体積の中身」ではなく、**界面（表面）の運動**  
- 多くのモデルは「表面が法線方向に移動」する枠組みに落とせる（表面運動）  
  - レベルセット法は、エッチング/デポの3Dプロファイル進化に広く使われてきた系譜がある  
  - 例：エッチングとデポのレベルセットモデリング（Adalsteinsson & Sethian）  
  - 例：DRIEをnarrow bandレベルセット＋MCで扱う3Dトポグラフィシミュレーション

### 4.3 SDF（Signed Distance Field）を状態とする利点
- **連続形状**（sub-voxel表現）  
- 法線 \(n=\nabla\phi/|\nabla\phi|\)、曲率 \(\kappa=\nabla\cdot n\) を計算可能  
- レベルセット方程式で安定更新でき、**トポロジ変化**に強い  
- **narrow band**により計算点数を削減（DRIEなどで実績）  

#### 参考（レベルセット／narrow band／半導体プロセス）
- ViennaPS：半導体プロセスのトポグラフィ進化に、**高性能レベルセット＋階層run-length encoding**を使用し、必要ならMCレイトレーシングも統合（2025）  
- DRIE：**narrow bandレベルセット**で表面進化、MCでフラックス分布を扱う3Dシミュレーション（2018）  
- 古典：エッチング/デポのレベルセット手法（1996）

## 5. “SDF narrow band” 詳細（実装に落ちる説明）

### 5.1 定義
- SDF：\(\phi(x)\) が「最も近い界面までの距離」、符号で内外を区別  
- narrow band：\(|\phi(x)|<w\) の点集合だけを保持（\(w\) は数voxel〜数十voxel）

### 5.2 narrow bandを使う理由（効果）
- 形状進化の計算（\(\phi_{t+\Delta t}\)）は \(\phi=0\) 近傍で決まる  
- 体積内部・外部の“遠方”は学習に不要（ほぼ一定）  
- 結果として  
  - **学習データ量**（点数）  
  - **推論計算量**（Sparse CNNの入力点数）  
  - **メモリ**  
  を削減できる

### 5.3 voxel→SDFの生成（典型）
- 3D occupancy（材料/空隙）から、内外の距離を別々に距離変換して符号付け  
- 重要：符号規約（内側が負/外側が正 など）を全工程で統一

### 5.4 再初期化（reinitialization）
レベルセット更新で \(\phi\) がSDFからずれることがあるため、定期的に距離場へ戻す。  
- 実装案（軽量→重い）
  1) 近傍再距離化（narrow band内のみの距離再計算）
  2) Fast Marching Method（Eikonal解）
  3) PDEベース再初期化（正確だが重い）  

Fast Marchingの解説・位置づけは古典教材にまとまっている。

## 6. 学習ターゲット：法線速度 \(V_n\)（閉包モデル）を学ぶ

### 6.1 レベルセット更新式（基本）
界面速度が法線方向 \(V_n\) のとき、レベルセットは
\[
\frac{\partial \phi}{\partial t} + V_n |\nabla \phi| = 0
\]
離散化すると
\[
\phi_{t+\Delta t} = \phi_t - \Delta t \, V_n \, |\nabla \phi_t|
\]

### 6.2 なぜ \(V_n\) が“閉包”になるのか
MCモデルでは、入射・反射・二次反応・拡散など多くの現象が重なって  
**最終的に界面がどのくらい進むか（速度）**として現れる。  
つまり、サロゲートとしては
- 内部の全事象を逐一再現するより  
- 「形状進化に効く統計」をまとめた \(V_n\) を学ぶ方が効率的

### 6.3 \(V_n\) の教師生成（SDF差分）
シミュレーション結果の \(\phi_t, \phi_{t+\Delta t}\) から、narrow band点で
\[
V_n(x,t) \approx \frac{\phi_t(x)-\phi_{t+\Delta t}(x)}{\Delta t \, |\nabla \phi_t(x)|}
\]
を作る（安定性のため \(|\nabla\phi|\) が小さい点は除外・クリップ）。  

> 重要：これにより **“1本の3D MC run” から表面点×時間の大量教師**が得られ、  
> run数（条件数）を爆増させなくても学習が成立しやすい。

## 7. Sparse CNN（疎畳み込み）採用の理由と設計

### 7.1 Dense 3D CNNのボトルネック
- 入力が 256³ のような格子だと、dense 3D convはコストが急増  
- しかし、narrow bandは全体の一部（薄いシェル）なので、dense処理は無駄が大きい

### 7.2 SparseTensorとして処理する発想
- non-zero（有効点）だけを座標＋特徴として持ち、畳み込みを行う  
- MinkowskiEngineはSparseTensorを基本表現にして疎畳み込みを提供している（ドキュメント明記）

### 7.3 入力特徴（推奨セット）
narrow band点ごとに：
- 幾何：\(\phi\), \(|\nabla\phi|\), 法線 \(n\), 曲率 \(\kappa\)  
- 材料/マスク：material_id, mask_id  
- 条件：レシピ要約 \(c\)（FiLM/concatで注入）  
- 潜在：\(z\)（未知フラックス/反応のまとめ）

### 7.4 ネットワーク形（推奨）
- Sparse U-Net（エンコーダ・デコーダ）  
- 条件注入：FiLM（ブロックごとにスケール・シフト）  
- 出力：\(V_n\)（スカラー）  
  - 拡張：\(V_\text{etch},V_\text{depo},V_\text{sputter}\) の成分分解も可能（原因分析が強くなる）

## 8. 蒸留（Teacher–Student）で“内部影響”を入れる

### 8.1 なぜ蒸留が必要か
現実の課題：
- 反射・再入射・遮蔽など「非局所」の影響は、表面幾何だけでは表現が難しい  
- MC内部には、直接フラックス・反射寄与・角度分布などの**豊富なログ**がある  
- しかし運用（実測合わせ）では内部ログは得られない

→ そこで、**学習時のみ内部ログを使い、運用時は不要**にする構成が合理的。

### 8.2 Teacher–Studentの役割分担
- **Teacher（学習専用）**：表面幾何＋レシピ＋（内部ログ） → \(V_n\) を高精度に学習  
- **Student（本番用）**：表面幾何＋レシピ＋潜在 \(z\) → \(V_n\) を再現  
  - Teacherの出力・中間表現を蒸留して、内部影響を“暗黙に”学ぶ

### 8.3 関連概念（理論的背景）
- LUPI（Learning Using Privileged Information）：学習時だけ特別な情報を使って学習を加速する枠組み  
- Knowledge Distillation：大きいモデル（またはアンサンブル）の知識を小さいモデルに移す

## 9. 同定（Calibration）：実測SEMに合わせて潜在 \(z\) を推定する

### 9.1 なぜ “潜在 \(z\)” を導入するのか
- フラックス計測が難しい  
- 表面反応のパラメータも直接測れない  
- しかし形状（SEM）には結果として現れる

→ 未知要素をすべて“物理パラメータ列挙”すると高次元で不安定。  
→ 形状に効く低次元に **圧縮した潜在 \(z\)** を同定対象にするのが現実的。

### 9.2 観測（SEM）をどう扱うか：特徴量化
まずは「画像を直接」ではなく、工学指標に落とすのが堅い。
- CD（高さ別）  
- sidewall angle  
- bottom曲率・フッティング指標  
- オーバーハング指標（あれば）

（将来拡張として、微分可能レンダリングで画像損失へ拡張可能。）

### 9.3 同定手段：MAP vs SBI
#### MAP（点推定）
\[
z^\* = \arg\min_z \| f_\text{feat}(\phi(z)) - y_\text{SEM}\|
\]
- 実装が簡単で速い  
- 多峰性（複数の \(z\) が同じ形状を作る）に弱い

#### SBI（Simulation-Based Inference：事後分布）
- 尤度が書けないブラックボックスシミュレータでも、  
  シミュレーションデータから \(p(z \mid y)\) を推定できる  
- `sbi` はSBIワークフローを提供し、amortized inferenceも可能（観測が変わっても再学習不要な範囲がある）

> 実務上のメリット：  
> 同定が“一点”ではなく“分布”になるため、  
> - 不確かさ（同定の信頼度）  
> - 反応/フラックスの非同定性  
> を明示でき、次の追加実験（能動学習）にも繋げやすい。

## 11. “同じ区分の類似手法”との比較（なぜこの組合せが良いか）

### 11.1 形状表現の比較
| 区分 | 方法 | 長所 | 短所 | 本手法が採用する理由 |
|---|---|---|---|---|
| 体積表現 | voxel occupancy | 単純、CNNが使える | 3Dで重い、連続性弱、微細丸みが難しい | 過去にデータ量爆増・運用コスト高になりやすい |
| 表面表現 | mesh/点群 | 表面だけで軽い | 位相変化/自己交差が難、再メッシュが面倒 | レベルセットの方が更新が堅い |
| **距離場** | **SDF（level-set）** | 連続、法線/曲率、位相変化に強い | 再初期化など数値管理が必要 | **時間発展・連続性担保の要件と一致** |
| implicit NN | Neural SDF | 高解像・連続 | 学習難、運用が重くなり得る | 本案はまず“グリッドSDF＋narrow band”で堅く |

### 11.2 モデル（ネットワーク）の比較
| 区分 | 方法 | 長所 | 短所 | 本手法が採用する理由 |
|---|---|---|---|---|
| dense 3D | 3D U-Net/RNN | 実装容易 | メモリ重、点数が膨大 | narrow bandに不適 |
| **sparse 3D** | **Sparse CNN** | narrow band点だけ計算、効率良 | 実装依存（SparseTensor） | **narrow bandと自然に整合** |
| surface graph | GNN/Transformer | 表面中心で軽い | 位相変化が難／非局所設計が要る | 将来拡張。まずSDFが堅い |
| neural operator | FNO/UNO等 | 解像度跨ぎ | 安定化が難、データ設計が要 | “Vn予測”補助として拡張可能 |

### 11.3 同定（実測吸収）の比較
| 区分 | 方法 | 長所 | 短所 | 本手法が採用する理由 |
|---|---|---|---|---|
| なし | 実測無視 | 実装簡単 | 実測に合わない | 要求と矛盾 |
| 点推定 | MAP最適化 | 速い、実装容易 | 多峰性に弱い | MVPとして有効 |
| **分布推定** | **SBI** | 尤度不要、多峰性OK、不確かさ | 実装がやや重い | **少数SEMでも“妥当範囲”が得られる** |

## 12. 得られるサロゲートモデルの仕様（想定I/O契約）

### 12.1 入力
- 初期形状：voxelまたはSDF（\(\phi_0\)）

## 13. 想定必要データ数（現実的な目安）

> 重要：本案は「run数＝学習サンプル数」ではない。  
> **1 runから表面点×時間の大量教師（Vn）が取れる**ため、run数を抑えやすい。

### 13.1 高忠実度シミュレーション（HF）
- **最小MVP**：30〜50 run  
  - 目的：同一形状カテゴリ内で「時間発展が破綻しない」レベル  
- **標準**：80〜150 run  
  - 目的：条件空間を広げても安定  
- **複数形状に展開**：形状ごと50〜100 run + 共有事前学習（転移）

### 13.2 1 runあたりのスナップショット数
- 20〜50 snapshot（多すぎるとI/Oがボトルネック）  
- 時間刻みは“物理時間”より“進行量（nm/step）”に揃えると学習が安定しやすい

### 13.3 SEM（実測）
- 与件：工程ごと10条件程度  
- 本案：  
  - 最初はMAPで latent同定（10でも回る可能性）  
  - 安定化したらSBI（事後分布）へ拡張  
  - 20〜30条件あると同定がより安定しやすい（ただし必須ではない）

## 14. 想定精度（何を“合っている”とするか）

精度は「SDFが一致」ではなく、実務KPIで評価する。

### 14.1 推奨評価指標（シミュレーション一致）
- SDF誤差（narrow band内）：MAE, Huber  
- 等値面の距離：Chamfer distance / Hausdorff（必要なら）  
- 工学指標：CD、sidewall angle、bottom形状（曲率）、オーバーハング指標

### 14.2 実測一致（SEM）
- 断面輪郭比較（pred vs SEM輪郭）  
- CD@複数高さ、角度、ボトムR、フッティング指標  
- 同定の不確かさ（SBIならcredible interval）

> “想定精度”は工程・解像度・SEMノイズに依存するため、本レポートでは数値断言を避け、  
> **「どの指標を何で保証するか」**を明確化することを優先する。

## 15. 想定課題とリスク（この組合せでも残るもの）

| リスク | 何が起きる | 代表的対策 |
|---|---|---|
| ロールアウト誤差蓄積 | 長時間で形状が崩れる | multi-step unroll loss、教師あり短窓、再初期化頻度調整 |
| SDFの数値劣化 | \(|\nabla\phi|\neq1\)、界面がぼける | Fast Marching/再距離化、更新スキームのCFL管理 |
| 潜在同定の非一意性 | 複数の \(z\) が同じ形状を作る | SBIで事後分布、事前分布で物理妥当域を制約、追加観測（別高さCD等） |
| OOD条件で暴走 | 学習範囲外で自信満々な予測 | OOD検知、UQ、能動学習で追加run |
| SEM→特徴の不安定 | 輪郭抽出がぶれる | セグメンテーションの安定化、特徴のロバスト化 |

## 16. 拡張性（次に何が足せるか：具体案）

### 16.1 多忠実度・能動学習
- HF（MC3D）を最小化し、LF（粗格子/簡略物理/2D断面）を大量に  
- 不確かさが高い条件だけHFを追加（Active Learning）

### 16.2 非局所性の強化（反射・再入射の表現）
- Teacher内部ログから、surface-to-surface結合（view-factor的）を学習し、Studentへ蒸留  
- あるいはSparse CNNに “非局所 attention” を追加

### 16.3 SEM画像を直接観測にする（観測モデル）
- 微分可能レンダリング（SDF→画像）を導入し、画像損失で同定  
- ただしまずは特徴量同定（MAP/SBI）が堅い

### 16.4 レシピ探索（最適化）
- サロゲートを高速評価器として、BO/MFBOで探索  
- 同定した潜在 \(z\) を条件として固定し、「装置ノブのみ探索」などが可能

## 17. 結論

本提案（1位案）は、次の理由で **与件（高速化 + 少データ同定）に最も整合**する：

1) **SDF + レベルセット更新**により、時間発展・連続性・位相変化に強い  
2) **narrow band + Sparse CNN**により、3D高解像でも計算資源を“表面近傍”に集中できる  
3) 学習ターゲットを **法線速度 \(V_n\)** にすることで、MCの複雑さを閉包として吸収し、run数を抑えやすい  
4) 学習時のみ内部ログを使う **Teacher–Student蒸留**で、二次反応等の“内部影響”を入れつつ運用を軽量に保てる  
5) 実測SEMが少数でも、未知を潜在 \(z\) に集約し **同定（MAP/SBI）**することで実測吸収が可能になる
